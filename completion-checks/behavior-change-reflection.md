Answer: I have not consistently done this yet, but I have started to think more about writing tests before prompts in general. I have started by identifying candidate prompts to start building out a more complete *prompt library* for my work.

B2: **Do you perform the load-bearing audit?** Not just on program exercises, but on any prompt you craft for real work?

Answer: I have not consistently done this yet, but as I have become more comfortable with EDD and Promptfoo, I have become more proficient at performing load-bearing audits. I plan to add this discipline to my workflow going forward.

B3: **Do you run the Model Ladder check?** When you build a new command or iterate on an existing one, do you run it against both a capable model and a lower-cost model? Have you actually adjusted any commands based on what the Haiku score told you?

Answer: The model-ladder checking I have found to be very useful in understanding many places where lower-cost models can be used to produce quality outputs. I have discovered more insights on when to use specific models based on the complexity of the task. By doing a better job of breaking up my workflows into smaller, more manageable tasks, I have been able to use lower-cost models more effectively.

Answer: No none of the artifacts from this program have become part of a real workflow yet. I have noticed over the last few weeks that I am starting to move from building prompts for a specific task to a mindset of building reusable prompts I can apply to similar tasks.

More and more it is front of my mind about building out a comprehensive prompt library for my work. The techniques, practice, and refinement of the artifacts produced in this program have given me a solid foundation to build upon.
